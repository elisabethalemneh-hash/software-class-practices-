# Software Class Practices

This repository is part of our Version Control Systems lab.  
It demonstrates how to use Git and GitHub for collaboration in a group project.

## Objectives
- Practice setting up a Git repository
- Commit changes from VS Code terminal
- Create branches using full names (e.g., hana-berhe-feature)
- Push branches to GitHub and open Pull Requests
- Collaborate as a team and merge contributions

## Lab Steps Practiced
1. Initialized repository with `git init`
2. Added and committed files (`git add .`, `git commit`)
3. Connected local repo to GitHub (`git remote add origin`)
4. Pushed changes (`git push`)
5. Created branches with full names
6. Opened Pull Requests and merged contributions
7. Invited lecturer to review the repository
